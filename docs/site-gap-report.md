## 高優先度の不足・改善点

### 🔴 favicon が存在しない
**現状**: `public/` ディレクトリにファイルが一切存在しない  
**基準**: favicon.ico または favicon.png が必須。ブラウザタブ・ブックマークに表示される  
**提案**: `public/favicon.ico`（16×16, 32×32）と `public/apple-touch-icon.png`（180×180）を追加し、`_document.tsx` の `<Head>` に `<link rel="icon" href="/favicon.ico" />` を追記する  
**実装難度**: 低

---

### 🔴 OGP 画像（og:image）が未設定
**現状**: `og:title` / `og:description` はあるが `og:image` がすべてのページで未設定  
**基準**: LINE・Twitter・Facebook でシェアされた際にサムネイル画像が表示されない  
**提案**: `public/ogp.png`（1200×630px）を作成し、各ページの `<Head>` に `<meta property="og:image" content="https://ドメイン/ogp.png" />` を追加。共通化は `src/pages/_app.tsx` の `DefaultSeo` または `_document.tsx` で行うと効率的  
**実装難度**: 低

---

### 🔴 robots.txt が存在しない
**現状**: `public/robots.txt` なし  
**基準**: 検索エンジンのクロール制御に必要。ないと意図しないページ（`/login` など）が検索結果に出る恐れがある  
**提案**: `public/robots.txt` を作成する

```
User-agent: *
Disallow: /login
Disallow: /api/
Allow: /
```

**実装難度**: 低

---

### 🔴 sitemap.xml が存在しない
**現状**: sitemap.xml なし  
**基準**: 検索エンジンへのページ認知速度に影響する  
**提案**: `next-sitemap` パッケージを導入する

```bash
npm install next-sitemap
```

`next-sitemap.config.js` を作成し、`package.json` の `build` スクリプトに `&& next-sitemap` を追記すると自動生成される  
**実装難度**: 低

---

### 🔴 プライバシーポリシーページがない
**現状**: お問い合わせフォームにて氏名・メールアドレスを収集しているが、プライバシーポリシーのページが存在しない  
**基準**: 個人情報を収集するフォームがあれば実質必須。フォーム下部に注記はあるが、ページとして独立したものが必要  
**提案**: `src/pages/privacy.tsx` を新規作成。フッターとフォーム下部にリンクを追加する  
**実装難度**: 低

---

### 🔴 お問い合わせフォームがメーラー起動のみで実際に送信されない
**現状**: `contact.tsx` の `handleSubmit` は `mailto:` URLを使ってメーラーを起動するだけ。サーバーには何も送信されない  
**基準**: ユーザーがメーラーを持っていない（スマートフォンのブラウザ等）場合に送信できない。高齢者ユーザーには特に問題  
**提案**: メール送信サービス（Resend / EmailJS / Formspree のいずれか）を導入する。最も簡単なのは [Formspree](https://formspree.io)（フリープランあり、Next.js と相性良し）または EmailJS  
**実装難度**: 中

---

### 🔴 カスタム 404 ページがない
**現状**: `src/pages/404.tsx` が存在しない。Next.js デフォルトの英語 404 ページが表示される  
**基準**: 高齢者ユーザーが迷子になったときに案内できない  
**提案**: `src/pages/404.tsx` を作成し、トップページへのリンクと主要ページへのナビゲーションを表示する  
**実装難度**: 低

## 中優先度の不足・改善点

### 🟡 避難場所・集会場の地図が埋め込まれていない
**現状**: `disaster.tsx` に避難場所の住所テキストと横浜市ハザードマップへの外部リンクはあるが、地図そのものがページ内に表示されない  
**基準**: 高齢者が外部サイトに移動せずに地図を確認できるべき  
**提案**: Google Maps Embed API（APIキー不要の基本埋め込み）または `<iframe>` で各避難場所の地図を表示する  
**実装難度**: 低〜中

---

### 🟡 役員・組織紹介ページがない
**現状**: 町内会長の連絡先が「回覧板参照」と記載されているのみ  
**基準**: 一般的な町内会サイトは役職名・担当者名・担当範囲を掲載している  
**提案**: `src/pages/activities.tsx` 内に役員一覧セクションを追加するか、専用ページを作成する。個人情報（電話番号等）は回覧板参照にとどめ、役職名と担当業務の掲載のみでも有効  
**実装難度**: 低

---

### 🟡 会費・加入案内ページがない
**現状**: 加入方法・会費額について説明するページもセクションも存在しない  
**基準**: 新転入者が「どうすれば入会できるか」を自力で調べられない  
**提案**: `src/pages/join.tsx` を作成し、加入方法・会費・問い合わせ先を掲載。ヘッダーナビまたはフッターにリンクを追加する  
**実装難度**: 低

---

### 🟡 テキストサイズ変更機能がない
**現状**: フォントサイズ調整ボタンがない  
**基準**: 高齢者ユーザーが多い地域サイトでは「文字を大きく」ボタンが推奨される  
**提案**: `Layout.tsx` にフォントサイズトグルボタンを追加し、`localStorage` でサイズ設定を保持する。または CSS の `font-size` を `rem` で統一し、ブラウザ側のズーム機能で対応できる旨をページ上部に案内する  
**実装難度**: 中

---

### 🟡 canonical タグが未設定
**現状**: 各ページに `<link rel="canonical" />` が設定されていない  
**基準**: 重複コンテンツと判定されるリスク（例：`http` / `https` の混在やトレイリングスラッシュ）がある  
**提案**: `next-seo` パッケージを導入し `DefaultSeo` で canonical を一括設定するか、各ページの `<Head>` に追加する  
**実装難度**: 低

---

### 🟡 構造化データ（JSON-LD）が未設定
**現状**: 構造化データがどのページにもない  
**基準**: `LocalBusiness` / `Organization` の構造化データにより Google 検索の「ナレッジパネル」表示や検索結果のリッチ化が期待できる  
**提案**: `src/pages/index.tsx` に以下を追加する

```tsx
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{ __html: JSON.stringify({
    "@context": "https://schema.org",
    "@type": "Organization",
    "name": "笠間町内会",
    "address": { "@type": "PostalAddress", "addressLocality": "横浜市栄区笠間" },
    "url": "https://ドメイン/"
  })}}
/>
```

**実装難度**: 低

---

### 🟡 アクセス解析ツールが未導入
**現状**: Google Analytics / Plausible 等の解析コードがない  
**基準**: どのページが閲覧されているか、スマートフォン比率などが把握できない  
**提案**: Vercel Analytics（デプロイ後に有効化、無料）か Google Analytics 4 を導入する。GA4 使用時は Cookie 同意バナーもセットで必要  
**実装難度**: 低〜中

---

### 🟡 実画像・写真素材が未設定
**現状**: `public/` が空で、各ページに実際の活動写真・地域の画像が一切ない  
**基準**: 活動の様子が伝わる写真があるかどうかで信頼感・閲覧継続率が大きく変わる  
**提案**: `public/images/` ディレクトリを作成し、`next/image` コンポーネントで最適化配信する。最低限トップのヒーローセクションと活動紹介ページに写真を追加する  
**実装難度**: 低（素材次第）

## 低優先度・あると良いもの

### 🟢 サイト内検索
**提案**: Algolia DocSearch（無料枠あり）または Pagefind（静的サイト向け OSS）の導入。情報量が増えてきた段階で検討  
**実装難度**: 中

### 🟢 ゴミカレンダーの印刷対応 CSS
**提案**: `@media print` スタイルでヘッダー・フッター・ナビを非表示にし、カレンダー部分だけを印刷できるようにする  
**実装難度**: 低

### 🟢 年間行事カレンダー（全件表示）
**提案**: `activities.tsx` または専用ページに月別・年間行事の一覧を掲載する。現状は「直近の予定」4件のみ  
**実装難度**: 低

### 🟢 地域関連リンク集ページ
**提案**: 横浜市・栄区役所・学校・消防署・ハザードマップ等をまとめたリンク集ページ（`src/pages/links.tsx`）  
**実装難度**: 低

### 🟢 SNS シェアボタン（LINE）
**提案**: お知らせページに LINE シェアボタンを追加。高齢者世代でも LINE 利用率が高い  
**実装難度**: 低

### 🟢 PDF 配布資料のダウンロード
**提案**: 回覧板・総会資料等を `public/docs/` に配置し、PDF ダウンロードリンクを各ページに設置  
**実装難度**: 低

### 🟢 Cookie 同意バナー
**提案**: GA4 等を導入した場合は必要。`react-cookie-consent` ライブラリが手軽  
**実装難度**: 低

### 🟢 RSS フィード
**提案**: お知らせの更新を RSS で受け取れるようにする。`feed` ライブラリ + Next.js API Route で実装可能  
**実装難度**: 中

### 🟢 会員専用ページ（回覧板デジタル化）
**提案**: `login.tsx` は既に存在するため、会員限定コンテンツ（回覧板 PDF・役員連絡先等）のホスティングに活用できる  
**実装難度**: 高

## 対応済みの良い点

- ✅ **トップページ構成**: ヒーロー・クイックリンク・最新お知らせ・直近予定・緊急連絡先がバランス良く配置されている
- ✅ **主要6ページ**: トップ・お知らせ・活動・ゴミ収集・防災・お問い合わせが揃っている
- ✅ **レスポンシブデザイン**: Tailwind の `md:` ブレークポイントで PC / スマートフォン両対応
- ✅ **title / meta description**: 全ページでページごとに固有の値が設定されている
- ✅ **OGP（og:title, og:description）**: 全ページで設定済み（og:image は未対応）
- ✅ **アクセシビリティ基礎**: `lang="ja"`, `aria-label`, `aria-current`, `aria-expanded`, `aria-labelledby` が適切に使用されている
- ✅ **セマンティック HTML**: `<header>`, `<nav>`, `<section>`, `<aside>`, `<main>` が構造的に使われている
- ✅ **防災情報ページ**: 緊急連絡先・避難場所一覧・備え・外部リンクが網羅されており充実度が高い
- ✅ **フォームのラベル対応**: `for` / `id` が紐づいており、スクリーンリーダー対応が取れている
- ✅ **緊急情報の視覚的差別化**: 赤系カラーで緊急情報が目立つようになっている
- ✅ **モバイルハンバーガーメニュー**: `aria-expanded` / `aria-controls` が正しく実装されている
- ✅ **Git 管理**: バックアップ・変更履歴が管理されている
- ✅ **Noto Sans JP フォント**: 日本語の可読性が高いフォントを採用
- ✅ **緊急連絡先の電話リンク**: `tel:` リンクでスマートフォンからワンタップ発信できる
