# 業界リサーチ：町内会・自治会ウェブサイト

**作成日:** 2026-06-07

---

## 1. 概要

町内会・自治会のウェブサイト事例と、住民が求める情報・機能を調査したもの。

---

## 2. 町内会ウェブサイトの現状

### 2.1 普及率の現状

- 日本の町内会・自治会のウェブサイト保有率は低く、多くの組織が回覧板・掲示板のみで運営
- デジタル化の波を受け、SNS（LINE・Facebook）を補助的に利用する組織が増加
- 公式ウェブサイトを持つ組織は主要都市の大規模自治会に限られる

### 2.2 よくある課題

| 課題 | 内容 |
|------|------|
| 更新頻度が低い | 担当者不在・スキル不足でコンテンツが古くなりやすい |
| モバイル非対応 | 高齢者のスマートフォン利用増加に追いついていない |
| 情報が分散 | 回覧板・掲示板・ウェブが並立し住民が混乱 |
| SEO 未対策 | 地域名で検索しても見つからない |

---

## 3. 住民が求める情報

利用者（住民）がウェブサイトに期待する情報の優先度：

| 優先度 | 情報 | 理由 |
|--------|------|------|
| 高 | ゴミ収集スケジュール | 毎週利用・最も問い合わせが多い |
| 高 | 緊急連絡先 | 災害時・急病時にすぐ参照したい |
| 高 | 避難場所 | 防災訓練・実際の避難時に必要 |
| 中 | お知らせ・イベント | 行事への参加判断に必要 |
| 中 | 役員・問い合わせ先 | 引越し時・困ったときの連絡先確認 |
| 低 | 活動紹介・沿革 | 関心があれば閲覧 |

---

## 4. 参考事例と設計方針

### 4.1 成功事例の共通点

- **シンプルなナビゲーション**：5〜6項目以内で全情報にアクセス可能
- **緊急情報の視認性**：防災・緊急連絡先は目立つ色（赤系）で配置
- **コンテンツの鮮度**：更新日付を表示し、情報の新鮮さを示す
- **外部リンクとの連携**：横浜市・区役所の公式情報へのリンクで補完

### 4.2 本プロジェクトへの適用

| 設計判断 | 理由 |
|---------|------|
| 6ページ構成に絞る | 管理コストを最小化し、更新頻度を維持しやすくする |
| 防災ページを赤系デザイン | 他ページと差別化し、緊急時に識別しやすくする |
| カテゴリーバッジ | お知らせの種別を視覚的に判断しやすくする |
| メーラー連携フォーム | サーバーレスでの問い合わせ受付（運用コスト削減） |
| kasama-green のブランドカラー | 地域・自然・安心感のイメージと整合 |

## 5. 技術スタック選定理由

| 技術 | 選定理由 |
|------|---------|
| Next.js | SSG による高速表示・SEO対応・TypeScript との親和性 |
| Tailwind CSS | 保守性が高く、カスタムカラーの追加が容易 |
| lucide-react | シンプルで統一感のあるアイコンライブラリ |
| メーラー連携 | バックエンドサーバー不要でコスト・運用を削減 |

---

## 6. 法規制・プライバシー

- 個人情報保護法：役員の個人名・電話番号をウェブに掲載しない（回覧板で案内）
- お問い合わせの個人情報はサーバー非経由のため、データ保存の法的リスクが低い
- 外部リンク先（横浜市等）のプライバシーポリシーは各サイトに準拠
