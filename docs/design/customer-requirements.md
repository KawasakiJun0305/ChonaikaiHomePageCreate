## 2. ビジネス背景

### 2.1 現在の課題

- 回覧板のみでの情報共有では配布に時間がかかり、緊急時の周知が遅れる
- ゴミ収集スケジュール・分別方法を問い合わせる住民が多く、役員の対応負担が大きい
- 転入してきた住民が町内会の活動・連絡先を知る手段がない
- 防災情報（避難場所・緊急連絡先）をいつでも確認できる手段がない

### 2.2 実現したいビジョン

住民が必要な情報をいつでも・どこからでも確認できる「地域の情報拠点」をウェブ上に作る。

### 2.3 期待される効果

- 役員への問い合わせ件数の削減（ゴミ収集・防災情報）
- 転入住民の町内会への参加・理解促進
- 緊急時（防災・行事中止等）の迅速な情報周知

## 3. 要求事項

### 3.1 機能要求

| # | 要求内容 | 優先度 |
|----|---------|--------|
| FR-1 | 最新のお知らせ・イベント情報を掲載できる | 必須 |
| FR-2 | ゴミ収集スケジュール・分別方法を掲載できる | 必須 |
| FR-3 | 避難場所・緊急連絡先など防災情報を掲載できる | 必須 |
| FR-4 | 活動紹介・年間行事カレンダーを掲載できる | 必須 |
| FR-5 | 住民からの問い合わせを受け付けられる | 必須 |

| FR-7 | スマートフォンから快適に閲覧できる | 必須 |

| # | カテゴリ | 要求内容 | 優先度 |

| NR-2 | 操作性 | 高齢者が使いやすいフォントサイズ・コントラスト | 重要 |

| NR-4 | 保守性 | 非エンジニアの役員がコンテンツを更新しやすい構造 | 重要 |
| NR-5 | パフォーマンス | ページ読み込みが3秒以内 | 重要 |

## 4. スコープ・制約

### 4.1 スコープ内

- トップページ・お知らせ・ゴミ収集・防災・活動紹介・お問い合わせの6ページ
- レスポンシブデザイン（PC・タブレット・スマートフォン対応）
- お問い合わせフォーム（メーラー連携方式）

### 4.2 スコープ外（現バージョン）

- 会員専用ログイン機能
- CMS（管理画面からのコンテンツ更新）
- 多言語対応
- 決済・会費徴収機能

### 4.3 制約

- **予算:** 最小限のホスティング費用に収める
- **技術:** 開発担当者が保守できる技術スタック（Next.js + TypeScript）
- **更新方法:** 現時点はコード編集でのコンテンツ更新
