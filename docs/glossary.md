## 技術用語・略語

---

### SSG

- **読み:** えすえすじー / Static Site Generation
- **カテゴリ:** 技術用語
- **説明:** ビルド時に全ページのHTMLを生成する手法。本プロジェクトでは `getStaticProps` を使用しない純粋な静的エクスポートを採用。コンテンツが静的でサーバーレス運用が可能なため選択された。
- **使用箇所:** `docs/design/basic-design.md`（レンダリング方式）
- **関連用語:** [ISR](#isr)
- **備考:** Next.js の `next export` で静的ファイルとして出力する

---

### ISR

- **読み:** あいえすあーる / Incremental Static Regeneration
- **カテゴリ:** 技術用語
- **説明:** ページを段階的に静的再生成する Next.js の機能。将来的に CMS 連携を追加する場合の移行候補として基本設計書に記載されている。現バージョンでは未使用。
- **使用箇所:** `docs/design/basic-design.md`（将来の拡張として言及）
- **関連用語:** [SSG](#ssg)

---

### OGP

- **読み:** おーじーぴー / Open Graph Protocol
- **カテゴリ:** 技術用語
- **説明:** SNS でページをシェアした際に表示されるタイトル・説明・画像を制御するメタタグの仕様。各ページで `og:title`・`og:description`・`og:type` を設定している。
- **使用箇所:** 全ページの `<Head>` 内（`og:title`, `og:description`, `og:type`）
- **関連用語:** [セマンティックHTML](#セマンティックhtml)

---

### WCAG AA

- **読み:** だぶりゅーしーえいじー えーえー / Web Content Accessibility Guidelines AA
- **カテゴリ:** 技術用語
- **説明:** ウェブアクセシビリティの国際標準規格 WCAG の AA 達成基準レベル。本プロジェクトではカラーコントラスト比のAAレベル準拠を要件としている。
- **使用箇所:** `docs/design/requirements.md`（2.2 アクセシビリティ）
- **関連用語:** [セマンティックHTML](#セマンティックhtml)、[モバイルファースト](#モバイルファースト)
- **備考:** 高齢者が使いやすいフォントサイズ・コントラストへの配慮（顧客要求書 NR-2）とも関連する

---

### モバイルファースト

- **読み:** もばいるふぁーすと / Mobile First
- **カテゴリ:** 技術用語
- **説明:** スマートフォン向けのデザイン・実装を優先し、そこから大きい画面（タブレット・PC）向けに拡張するアプローチ。Tailwind CSS のブレークポイント（`sm` / `md` / `lg`）を使って実装。
- **使用箇所:** `docs/design/requirements.md`（2.1 レスポンシブデザイン）
- **関連用語:** [WCAG AA](#wcag-aa)

---

### セマンティックHTML

- **読み:** せまんてぃっくえいちてぃーえむえる / Semantic HTML
- **カテゴリ:** 技術用語
- **説明:** `<header>`・`<main>`・`<section>`・`<article>` など、意味を持つ HTML 要素を適切に使用すること。アクセシビリティ・SEO 向上のために要件として定められている。
- **使用箇所:** `docs/design/requirements.md`（2.2 アクセシビリティ）
