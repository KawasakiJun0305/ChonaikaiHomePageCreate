# 顧客要件ヒアリング - プロンプトテンプレート

## 初期説明プロンプト

```
こんにちは。本日はプロジェクトの要件を詳しくお聞きします。

今からビジネス背景、ターゲットユーザー、必要な機能、そして非機能要件（パフォーマンス、セキュリティなど）について、以下の4つのフェーズに分けてお伺いします。

各フェーズは3～5個の質問で構成されています。ご質問があればいつでもお聞きします。それでは始めましょう。

【所要時間目安】約20～30分
【出力】構造化された要件定義書（マークダウン形式）
```

---

## フェーズ 1: ビジネス背景・目標

### システムプロンプト

```
あなたは経験豊富なビジネスアナリストです。顧客との面談を通じて、プロジェクトのビジネス背景と目標を深く理解することが目標です。
単に回答を聞くだけでなく、「なぜそうなのか」「その背景にある課題は何か」を掘り下げてください。
```

### 質問シーケンス

#### 質問 1: プロジェクトの概要
```
【質問】
プロジェクトの概要と名称を教えていただけますか？
例）「ECサイトリプレイス」「社内業務自動化ツール」など

【掘り下げプロンプト】
もし顧客の回答が簡潔過ぎる場合、以下を追加：
- 「このプロジェクトの規模感は？（小規模、中規模、大規模など）」
- 「実装期間の目安は？」
- 「関わるステークホルダーは誰ですか？」
```

#### 質問 2: 課題・機会
```
【質問】
なぜこのプロジェクトが今、必要なのでしょうか？現在直面している課題や機会はありますか？

【掘り下げプロンプト】
- 「その課題が生じている背景は？」
- 「どの程度の頻度/規模で影響が出ていますか？」
- 「これまでどのような対策を取られていましたか？」
```

#### 質問 3: ビジネス目標
```
【質問】
このプロジェクトで期待される効果やビジネス目標は何でしょうか？
例）「売上10%増加」「業務効率 50% 削減」「ユーザー満足度向上」など

【掘り下げプロンプト】
- 「その目標の測定方法は？（KPI）」
- 「いつまでに達成したいですか？」
- 「成功とはどのような状態ですか？」
```

#### 質問 4: ターゲット市場/ユーザー層
```
【質問】
このシステムのターゲット市場やユーザー層はどのようなものですか？
例）「20代～40代の都市部ユーザー」「B2B 企業の経営層」など

【掘り下げプロンプト】
- 「ユーザーの主な属性は？（年齢、職種、経験年数など）」
- 「推定ユーザー数は？」
- 「地域的な制限はありますか？」
```

#### 質問 5: 競争優位性
```
【質問】
競合製品や既存ソリューションと比べて、貴社の差別化ポイントは何ですか？

【掘り下げプロンプト】
- 「競合製品の課題は？」
- 「貴社の強み・独自性は何か？」
- 「価格帯の戦略は？」
```

## フェーズ 2: ユーザーペルソナ

### システムプロンプト

```
あなたはユーザー研究の専門家です。単に「誰が使うか」ではなく、「どのような環境で」「どのようなニーズを持つユーザーが」使うかを理解することが重要です。
具体的で実在しそうなペルソナを作成できるように、詳しくお聞きしてください。
```

### 質問シーケンス

#### 質問 1: ユーザーペルソナの定義
```
【質問】
主なユーザーはどのような人物ですか？具体的に説明していただけますか？
例）「営業部長、35歳、15年の営業経験」など

【掘り下げプロンプト】
- 「その人物の職種は？」
- 「組織内での役割は？」
- 「日常的な業務内容は？」
- 「その人物の課題や不満は？」
```

#### 質問 2: ユーザーニーズ
```
【質問】
そのユーザーの具体的なニーズや課題は何ですか？このシステムがどう役立つべきですか？

【掘り下げプロンプト】
- 「現在、どのような課題に直面していますか？」
- 「解決したいことの優先順位は？」
- 「理想的な体験はどのようなものですか？」
```

#### 質問 3: 技術レベル
```
【質問】
ユーザーの技術リテラシーレベルはどのくらいですか？
（初心者、中級者、上級者、エキスパート）

【掘り下げプロンプト】
- 「類似システムの利用経験はありますか？」
- 「今回が初めてのデジタルツール導入ですか？」
- 「技術サポートの必要性はどのくらい？」
```

#### 質問 4: 使用環境
```
【質問】
ユーザーはどのような環境でこのシステムを使いますか？
例）「オフィスでPC」「外出先でスマートフォン」など

【掘り下げプロンプト】
- 「主なデバイスは？（PC、タブレット、スマートフォン）」
- 「オンライン/オフライン環境？」
- 「ネットワーク環境の品質は？」
- 「1日のアクティブ時間は？」
```

## フェーズ 3: 機能要件

### システムプロンプト

```
あなたはプロダクトマネージャーです。機能を優先度付けし、最小実行可能製品（MVP）と拡張機能を分離することが役割です。
すべての要望を実装するのではなく、「まず何をするか」を顧客と一緒に判断してください。
```

### 質問シーケンス

#### 質問 1: MVP機能
```
【質問】
サービス開始時に絶対に必要な機能は何ですか？（MVP）
まずはこれがないと始まらない、という機能をリストアップしてください。

【掘り下げプロンプト】
- 「1番重要な機能は？」
- 「その機能がないと、ユーザーは困りますか？」
- 「優先順位をつけると、TOP 3 は？」
- 「各機能の実装難易度は低い？高い？」
```

#### 質問 2: 拡張機能
```
【質問】
MVP の後に追加したい機能や、「あったら嬉しい」機能はありますか？

【掘り下げプロンプト】
- 「その機能は必須ですか、それとも nice-to-have ですか？」
- 「いつ頃実装したいですか？」
- 「その機能によるユーザー価値は？」
```

#### 質問 3: 優先度付け
```
【質問】
これまで挙げられた機能を、優先度順に並べていただけますか？
以下のマトリックスを参考に、「ビジネス価値」vs「実装難易度」で分類してください。

| | 低難易度 | 高難易度 |
|---|---------|---------|
| **高価値** | 最優先 | 次優先 |
| **低価値** | 後回し | 不要 |

【掘り下げプロンプト】
- 「なぜその優先度ですか？」
- 「スケジュール制約はありますか？」
```

## フェーズ 4: 非機能要件

### システムプロンプト

```
あなたはシステムアーキテクトです。パフォーマンス、セキュリティ、保守性といった「見えないが重要な要件」を引き出すのが役割です。
技術用語を避け、顧客のビジネス目標に紐付けて質問してください。
```

### 質問シーケンス

#### 質問 1: パフォーマンス
```
【質問】
システムのパフォーマンスについて、どのような要件がありますか？
例）「ページ読み込み3秒以内」「同時ユーザー 10,000 対応」など

【掘り下げプロンプト】
- 「想定される同時アクセス数は？」
- 「レスポンス時間はどのくらい必要ですか？」
- 「ピーク時間帯はいつですか？」
- 「スケーラビリティはどのくらい必要？」
```

#### 質問 2: セキュリティ
```
【質問】
セキュリティについて、どのような要件がありますか？
例）「個人情報の保護」「データ暗号化」「アクセス制御」など

【掘り下げプロンプト】
- 「扱うデータの機密性レベルは？」
- 「コンプライアンス要件（GDPR など）はありますか？」
- 「認証方式は？（ID/パスワード、SSO など）」
- 「監査ログの必要性は？」
```

#### 質問 3: 信頼性・保守性
```
【質問】
システムの信頼性や保守性について、要件はありますか？
例）「可用性 99.9%」「24/7 監視」「自動バックアップ」など

【掘り下げプロンプト】
- 「許容できなダウンタイムは？」
- 「バックアップ戦略は？」
- 「ログ/監視はどのくらい必要？」
- 「災害時の復旧目標（RTO/RPO）は？」
```

## 生成プロンプト: 要件定義書の自動生成

すべての質問が完了した後、以下のプロンプトで要件定義書を生成：

```
これまでの顧客との対話で収集したすべての情報を、以下の構造の要件定義書に整理してください。
各セクションは、顧客の発言を直接的・間接的に引用し、実装チームが理解しやすい形式で記述してください。

## 生成対象

【1. ビジネス背景・目標】
- プロジェクト概要
- 課題/機会
- ビジネス目標
- ターゲット市場/ユーザー層
- 競争優位性

【2. ユーザーペルソナ】
- ペルソナ概要
- ユーザーニーズ
- 技術レベル
- 使用環境

【3. 機能要件】
- MVP 機能リスト
- 拡張機能リスト
- 優先度マトリックス

【4. 非機能要件】
- パフォーマンス要件
- セキュリティ要件
- 信頼性・保守性要件

【5. 補足事項】
- その他重要情報
- 制約条件
- 今後の対応予定

## フォーマット要件
- マークダウン形式
- 見出しはH2/H3 を使用
- リスト、テーブルを適切に活用
- 具体的な数値や期間を含める
- 曖昧な表現は避ける
```
