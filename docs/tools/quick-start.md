# 顧客要件ヒアリングスキル - クイックスタート

環境を汚さない、最小限のセットアップで顧客要件を効率的に引き出すスキルです。

---

## 📖 使い方

### 1. スキルを実行

Claude Code のターミナルで以下を実行：

```bash
/customer-req
```

### 2. ガイダンスに従う

Claude が以下の4つのフェーズを順番にガイドします。各質問に答えるだけです：

#### **フェーズ 1: ビジネス背景・目標** (5項目)
- プロジェクトの概要と名称
- なぜこのプロジェクトが必要か（課題・機会）
- 期待される効果・ビジネス目標
- ターゲット市場・ユーザー層
- 競合製品との差別化ポイント

#### **フェーズ 2: ユーザーペルソナ** (4項目)
- 主なユーザーの人物像
- ユーザーの具体的なニーズ
- ユーザーの技術レベル
- 使用環境・デバイス

#### **フェーズ 3: 機能要件** (3項目)
- 必須機能（MVP）
- 将来追加したい機能
- 優先度付け

#### **フェーズ 4: 非機能要件** (3項目)
- パフォーマンス要件
- セキュリティ要件
- 信頼性・保守性要件

### 3. 要件定義書を受け取る

すべての質問に回答すると、Claude が **構造化された要件定義書** を自動生成し、以下のディレクトリに保存します：

```
outputs/generated-requirements/
└── [プロジェクト名]_要件定義書.md
```

---

## ⏱️ 所要時間

**約 20～30 分**

---

## 📋 出力形式

生成される要件定義書は以下の構成：

```markdown
# プロジェクト要件定義書

## 1. ビジネス背景・目標
- プロジェクト概要
- 課題/機会
- ビジネス目標
- ターゲット市場/ユーザー層
- 競争優位性

## 2. ユーザーペルソナ
- ペルソナ概要
- ユーザーニーズ
- 技術レベル
- 使用環境

## 3. 機能要件
- MVP（最小実行可能製品）
- 将来実装予定機能
- 優先度マトリックス

## 4. 非機能要件
- パフォーマンス
- セキュリティ
- 信頼性・保守性

## 5. 補足事項
```

---

## 💡 ヒント

### 質問に困ったら

各質問に対して、Claude は自動的に **掘り下げ質問** を提供します。簡潔な回答でもかまいません。Claudeが詳細を引き出します。

### 複雑なプロジェクトの場合

プロジェクトが複雑な場合は、複数回に分けることをお勧めします：
1. 最初のセッションで MVP だけに絞る
2. 後のセッションで拡張機能や詳細を追加

### 既存プロジェクトの場合

既に決まっている要件がある場合は、その内容をそのまま答えても大丈夫です。Claude が整理してくれます。

## 環境の清潔さについて

このテンプレートは以下の考慮で設計されています：

- ✅ 生成物は `outputs/` に集約（プロジェクトルート内で整理）
- ✅ `.gitignore` で生成物を除外（Git リポジトリを汚さない）
- ✅ 最小限の依存関係（PCに新しいソフトをインストール不要）
- ✅ スキル単体での使用を優先（Next.js 開発は必須ではない）

## よくある質問

**Q: このテンプレートを別のプロジェクトに組み込める？**

A: `outputs/generated-requirements/` ディレクトリと `customer-requirements-*.md` ファイルだけ必要です。他のファイルは選択的に使用できます。

**Q: 生成された要件定義書を編集できる？**

A: もちろんです。マークダウン形式なので、任意のエディタで編集可能です。

**Q: スキルを繰り返し実行できる？**

A: はい。複数のプロジェクト用に複数の要件定義書を生成できます。ファイル名が異なれば、`outputs/` 内に並存します。
