# 顧客要件ヒアリングスキル

## スキル概要
顧客の立場に立って、新規プロジェクトに対する要件をインタラクティブに引き出し、構造化された要件定義書を自動生成するスキルです。

## 目的
- 顧客との対話を通じてプロジェクトの背景、目標、ユーザー情報、機能・非機能要件を体系的に収集
- 実装チームが使用可能な詳細な要件定義書を生成
- プロジェクト初期段階でのコミュニケーションギャップを軽減

## 動作フロー

### フェーズ 1: ビジネス背景・目標 （5項目）
```
1. プロジェクトの概要/名称は？
2. なぜこのプロジェクトが必要か？（課題や機会）
3. 期待される効果/ビジネス目標は？
4. ターゲット市場/ユーザー層は？
5. 競合製品や既存ソリューションとの違いは？
```

### フェーズ 2: ユーザーペルソナ （4項目）
```
1. 主なユーザーは誰か？（職種、スキルレベルなど）
2. ユーザーの課題/ニーズは何か？
3. ユーザーの技術レベルは？（初心者〜エキスパート）
4. 使用環境/デバイスは？（Web, Mobile, Desktop, など）
```

### フェーズ 3: 機能要件 （3項目）
```
1. 必須機能（MVP）は何か？
2. 将来追加したい機能は？
3. 優先度付けはどうするか？
```

### フェーズ 4: 非機能要件 （3項目）
```
1. パフォーマンス要件は？（レスポンス時間、スケーラビリティなど）
2. セキュリティ要件は？（認証、データ保護など）
3. 信頼性・保守性要件は？（可用性、監視、ログなど）
```

## 使用方法

1. ユーザーが `/customer-req` コマンドを実行
2. Claude がフェーズ1からフェーズ4まで順番に質問を提示
3. ユーザーが各質問に対して回答
4. すべての質問に答えた後、自動的に要件定義書が生成される

## 実装の詳細

### プロンプトテンプレート

各フェーズの開始時に以下のようなプロンプトを実行：

```
あなたは経験豊富なビジネスアナリストです。顧客から詳細な要件を引き出す役割を担当しています。
以下の項目について、顧客に段階的に質問してください。各質問では顧客の回答を詳しく引き出すようにしてください。

【現在のフェーズ】
{フェーズ名}

【質問項目】
{質問リスト}

【進捗】
{現在の進捗状況}

ユーザーの回答を受けて、以下を実行してください：
1. 回答内容を確認し、必要に応じて追加の掘り下げ質問をする
2. 次のフェーズへ進む判断をする
```

---

## カスタマイズ可能な部分

- 質問項目の数や内容
- 出力フォーマット
- 質問の詳細度（簡潔 vs 詳細）
- ドメイン別の質問テンプレート（B2B, B2C, etc）

---

## 注意事項

- このスキルは初期要件の収集用です。詳細な仕様書は別途作成が必要
- 複雑なプロジェクトの場合、複数回のセッションに分けることを推奨
- 顧客の回答が不十分な場合は、追加の掘り下げ質問を自動的に実施
